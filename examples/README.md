# Examples

Runnable examples on real data — daily AAPL prices, 2020–2024, committed in
`data/AAPL.csv`, so everything runs offline.

Each script is self-contained and takes no arguments: it loads the prices and
calls the library directly, so you can read any one top to bottom. To try a
different model, change the one line that builds it. Output PNGs/PDFs are written
to `output/` (created on first run).

## The full backtest

`full_backtest.py` is the "give it data, get a report" demo. It loads the prices,
calls `run_backtest`, and saves a single labelled dashboard page (PNG and PDF) —
no flags. The title and the metrics footer are generated by the library:

```bash
python examples/full_backtest.py
```

To backtest a different model, change the one line that builds it, or call
`varlib.run_backtest(model, prices=..., window=...)` yourself — see the top-level
README.

## Individual charts

Each script in `charts/` renders exactly one plot, so you can look at them in
isolation:

| Script                       | Chart                                            |
|------------------------------|--------------------------------------------------|
| `charts/breaches.py`         | VaR forecast vs realised loss, breaches in red.  |
| `charts/timeline.py`         | Timeline of breach days, to reveal clustering.   |
| `charts/traffic_light.py`    | Basel green/yellow/red zones with the count.     |
| `charts/distribution.py`     | Return histogram with the VaR and ES lines.      |

```bash
python examples/charts/breaches.py
python examples/charts/timeline.py
python examples/charts/traffic_light.py
python examples/charts/distribution.py
```

## Console-only

`single_instrument.py` is the end-to-end workflow with no charts: estimate VaR
with every model on the last two years, then roll the Historical model through
the full history and print the backtest. Needs only numpy + pandas.

```bash
python examples/single_instrument.py
```
